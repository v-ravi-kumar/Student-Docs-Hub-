# Student Document Access & Management System (SDAMS) 🎓

SDAMS is a modern, web-based centralized portal designed to eliminate the manual, time-consuming processes associated with student documentation. 
It provides a secure, 24/7 digital platform for students to access and download their academic documents (Marksheets, Transfer Certificates, etc.) while offering college administrators a powerful dashboard to securely manage and upload student files.

## 🚀 Features

### For Office Staff (Admin)
- **Secure Login:** Role-protected access.
- **Student Registration:** Ability to register new students onto the system.
- **Document Management:** Upload PDFs and images (Marksheets, TCs, Degree Certificates) linked directly to the respective student.
- **Student Overview:** View a list of registered students.

### For Students
- **Instant Access:** Access the portal 24/7—no more waiting for office hours.
- **Dashboard:** Modern, glassmorphism-styled UI displaying all uploaded documents.
- **Secure Downloads:** One-click downloads for academic and credential documents.

---

## 🛠 Technology Stack

This project was built with a split frontend/backend architecture.

**Frontend:**
- **React.js** (Bootstrapped with Vite)
- **React Router** for seamless SPA navigation
- **Axios** for API requests
- **Lucide-React** for premium SVG icons
- **CSS** (Custom glassmorphism design for a premium UI feel)

**Backend:**
- **Python (Flask)**
- **Flask-SQLAlchemy** (Database ORM mapping)
- **Flask-JWT-Extended** (Secure authentication & token management)
- **Flask-CORS** (Cross-origin resource sharing)
- **MySQL / SQLite** (Defaults to SQLite for local development convenience)

## 💻 Installation & Setup

### Prerequisites
Make sure you have installed on your local machine:
- [Node.js](https://nodejs.org/) (v16+)
- [Python](https://www.python.org/downloads/) (v3.9+)

### 1. Clone the Repository
If you're pulling from GitHub, clone the repo. Otherwise, navigate to the `sdams` project folder.

```bash
cd sdams
```

### 2. Backend Setup
The backend runs on a Python Flask server attached to a local database.

1. Navigate to the `backend` directory:
   ```bash
   cd backend
   ```
2. Create and activate a Virtual Environment (Recommended):
   ```bash
   # On Windows
   python -m venv venv
   venv\Scripts\activate
   ```
3. Install the dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Initialize the database and create the root Admin user:
   ```bash
   python init_db.py
   ```

### 3. Frontend Setup
The frontend runs on a Vite-powered React development server.

1. Navigate to the `frontend` directory:
   ```bash
   cd frontend
   ```
2. Install Node modules:
   ```bash
   npm install
   ```

## 🏃‍♂️ Running the Application

To run the Student Document Access & Management System (SDAMS) locally, you must start both the **Backend (Flask API)** and the **Frontend (React)** development servers simultaneously.

### 1. Start the Backend Server (Flask)
The backend manages the database, authentication, and document storage.

1. Open a new Terminal (PowerShell or CMD).
2. Navigate to the `backend` directory.
3. Activate the virtual environment and start the server:

**Windows PowerShell (Standard):**
```powershell
cd backend
.\venv\Scripts\activate
python app.py
```

> [!TIP]
> **If scripts are disabled (Execution Policy Error):**
> Use the following command to bypass the policy for the current session:
> `powershell -ExecutionPolicy ByPass -File venv\Scripts\Activate.ps1`
> 
> *Alternatively, run directly without activation:*
> `.\venv\Scripts\python.exe app.py`

**Linux/macOS:**
```bash
cd backend
source venv/bin/activate
python app.py
```

*The Flask API will start on **http://127.0.0.1:5000**.*

---

### 2. Start the Frontend Server (Vite/React)
The frontend provides the user interface for both students and administrators.

1. Open a **second** Terminal window.
2. Navigate to the `frontend` directory.
3. Start the development server:

**Standard Command:**
```bash
cd frontend
npm run dev
```

> [!TIP]
> **If "npm" command is blocked in PowerShell:**
> Run it via the Command Prompt wrapper:
> `cmd /c npm run dev`

*The Vite development server will start on **http://localhost:5173** (or the port shown in your terminal).*
